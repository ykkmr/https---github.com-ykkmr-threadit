# 🔄 Asynchronous Data Update Strategy

본 문서는 프론트엔드(Vue 3)와 백엔드(Node.js) 간의 상태를 동기화하기 위한 비동기 갱신 전략 및 주기 설정 가이드라인입니다. 

기능의 **실시간성 요구 수준**과 **서버 리소스 비용**을 트레이드오프(Trade-off)하여 세 가지 방식 중 하나를 선택해 구현합니다.



---

## 1. 타이머 기반 폴링 (HTTP Polling / Short Polling)
`setInterval` 등을 사용하여 클라이언트가 서버에 주기적으로 데이터를 요청하는 방식입니다.

### 📌 적용 대상 (Low to Medium Priority)
- **트렌딩 해시태그 목록:** 1분 ~ 5분 주기로 갱신 (실시간성이 크게 중요하지 않음).
- **스레드/패치 누적 조회수:** 30초 ~ 1분 주기로 갱신.
- **오프라인 유저의 부재중 알림 확인:** 1분 주기로 갱신.

### ⚠️ 구현 룰 (Vue 3 & 브라우저 최적화)
1. **갱신 주기 (Interval):** 기본적으로 최소 `10초` 이상의 간격을 둡니다. (1초 단위 폴링 금지)
2. **Page Visibility API 적용:** 사용자가 브라우저 탭을 내리거나 다른 탭을 보고 있을 때(`document.hidden === true`)는 **타이머를 일시 정지**하여 불필요한 서버 API 호출을 100% 차단해야 합니다.
3. **컴포넌트 언마운트 시 해제:** 컴포넌트가 파기될 때 `onUnmounted` 훅에서 반드시 `clearInterval()`을 호출하여 메모리 누수(Memory Leak)를 방지합니다.

---

## 2. 단방향 스트리밍 (Server-Sent Events, SSE)
클라이언트가 한 번 연결을 맺으면, 서버에서 이벤트가 발생할 때마다 클라이언트로 데이터를 밀어주는(Push) 단방향 통신 방식입니다. 무거운 웹소켓보다 가볍고 HTTP/2와 궁합이 좋습니다.

### 📌 적용 대상 (High Priority, Read-Only)
- **스레드 실시간 피드:** 누군가 내 글에 새로운 답글(Reply)을 달았을 때 즉각적인 알림 및 렌더링.
- **좋아요 카운트 실시간 반영:** 특정 스레드를 보고 있을 때 좋아요 수가 올라가는 연출.
- **AI 봇(ClawdBot) 타이핑 상태:** 봇이 답변을 생성 중일 때 "ClawdBot이 입력 중입니다..." 상태 표시.

### ⚠️ 구현 룰
- 연결이 끊어졌을 때 브라우저가 자동으로 재연결을 시도하므로, 백엔드에서는 클라이언트의 재연결 요청을 부드럽게 처리할 수 있도록 설계해야 합니다. (이전 MOFIN 테스트에서 구현했던 로직 활용)

---

## 3. 양방향 실시간 통신 (WebSockets / Socket.io)
클라이언트와 서버가 지속적인 연결(Persistent Connection)을 맺고 양방향으로 데이터를 매우 빠르게 주고받는 방식입니다. 서버 리소스를 가장 많이 소모합니다.

### 📌 적용 대상 (Ultra-High Priority, Interactive)
- **다중 유저 실시간 잼 세션 (Co-op Synth):** 여러 사용자가 하나의 모듈러 신디사이저 패치에 접속해 동시에 노브를 돌리고 케이블을 연결할 때 (0.1초 단위의 지연도 없어야 함).
- **1:1 실시간 다이렉트 메시지 (DM):** 사용자들이 채팅 룸에서 대화를 나눌 때.

### ⚠️ 구현 룰
1. **지수 백오프 (Exponential Backoff):** 네트워크 불안정으로 소켓이 끊어졌을 때, 1초, 2초, 4초, 8초 식으로 재연결 시도 간격을 늘려 서버의 순간적인 과부하(Thundering Herd)를 막아야 합니다.
2. **Room / Channel 분리:** 전체 브로드캐스트를 피하고, 반드시 특정 스레드 ID나 패치 ID를 기반으로 생성된 `Room` 단위로만 이벤트를 Emit하여 트래픽을 격리합니다.

## 📊 요약: 갱신 주기 결정 매트릭스 (Decision Matrix)

| 기능 분류 | 권장 기술 | 갱신 주기 / 조건 | 데이터 방향 |
| :--- | :--- | :--- | :--- |
| **통계/트렌드** | HTTP Polling | 1분 ~ 5분 단위 (탭 활성화 시에만) | 클라이언트 ↔ 서버 |
| **피드/알림/답글**| SSE (Server-Sent Events) | 서버에서 이벤트 발생 시 즉시 | 서버 ➔ 클라이언트 |
| **라이브 합주/채팅**| WebSocket (Socket.io) | 액션 발생 시 실시간 (ms 단위) | 클라이언트 ↔ 서버 |
